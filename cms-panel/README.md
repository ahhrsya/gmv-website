# GMV CMS Panel

A standalone CMS that edits `content/home.md` in the `ahhrsya/gmv-website` GitHub repo directly from the browser — no server required.

## How it works

The panel is a **static HTML/JS site** (deployable to Cloudflare Pages, Netlify, Vercel, or any static host) that calls the **GitHub Contents API** directly from the browser using a user-supplied **GitHub Personal Access Token (PAT)**.

1. User opens the panel URL
2. Pastes their GitHub PAT (stored in `localStorage`)
3. Reads `content/home.md` from the repo (base64 decoded, YAML frontmatter parsed)
4. User edits bilingual fields (EN + ID) in the browser
5. On Publish → PUT request to GitHub Contents API with new base64-encoded content
6. GitHub commits to `main` branch → Vercel auto-deploys → site is live in ~60s

## Files

- `public/index.html` — UI shell
- `public/app.js` — all logic (GitHub API calls, frontmatter parser, bilingual editor)
- `public/styles.css` — styling
- `src/worker.js` — *(optional)* Cloudflare Worker for any server-side needs
- `wrangler.toml` — *(optional)* Cloudflare Pages deploy config

## GitHub PAT requirements

Create a PAT at https://github.com/settings/tokens with these scopes:

- ✅ `repo` (full repository access) — required to read & write file contents

Store it in the panel's token input field on first use. It persists in `localStorage` and is never sent to any server (the panel is fully static).

## Deploying to Cloudflare Pages

```bash
cd cms-panel
npx wrangler pages deploy public --project-name=gmv-cms
```

## Local development

```bash
cd cms-panel/public
python3 -m http.server 8000
# open http://localhost:8000
```
